> **Mục tiêu:** S3 Static Website Hosting, Versioning, Lifecycle Rules, Cross-Region Replication.

## S3演習 1-1: Static Website Hosting + Bucket Policy

### Bước 1: Tạo S3 Bucket

| Mục | Giá trị |
|-----|---------|
| Tên Bucket | `lab3-[クラス名+番号]-bucket` |
| Region | us-east-1 |
| Block all public access | ❌ **Tắt** (Unchecked) |

### Bước 2: Upload file

1. Upload `index.html` và các file hình ảnh lên Bucket
2. Upload xong — mặc định file không public

### Bước 3: Bật Static Website Hosting

1. **Properties → Static website hosting → Edit**
2. Enable → Index document: `index.html`
3. Ghi lại **Bucket website endpoint** (dạng `http://bucket-name.s3-website-us-east-1.amazonaws.com`)

### Bước 4: Cấu hình Bucket Policy (cho phép public read)

**S3 → Permissions → Bucket policy → Edit:**

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::lab3-[クラス名+番号]-bucket/*"
    }
  ]
}
```

> ⚠️ Phải **tắt "Block all public access"** trước khi áp dụng policy này.

### Bước 5: Kiểm tra

Truy cập **Bucket website endpoint** → trang web hiển thị ✅

## S3演習 1-2: Versioning

### Bước 1: Bật Versioning

**S3 → Properties → Bucket Versioning → Enable**

### Bước 2: Test Versioning

1. Upload một file (vd: `index.html`) lần đầu
2. Sửa nội dung → upload lại cùng tên file
3. **S3 → Objects → "List versions"** → thấy 2 versions ✅
4. Có thể **khôi phục** version cũ bằng cách xóa version mới nhất

### Bước 3: Xóa file và khôi phục

1. Xóa file → S3 tạo **Delete marker** (không xóa thật)
2. Vào **Show versions** → xóa Delete marker → file phục hồi ✅

## S3演習 1-3: Lifecycle Rules + Cross-Region Replication

### Bước 1: Lifecycle Rules

**S3 → Management → Create lifecycle rule**

**Rule 1:** `[クラス名+番号]_lcy_rule1`

| Mục | Giá trị |
|-----|---------|
| Scope | All objects in bucket |
| Action | Transition **noncurrent versions** |
| Storage class | **S3 Standard-IA** |
| Days after becoming noncurrent | **30 ngày** |

**Rule 2:** `[クラス名+番号]_lcy_rule2`

| Mục | Giá trị |
|-----|---------|
| Scope | All objects in bucket |
| Action | **Expire noncurrent versions** |
| Days after becoming noncurrent | **365 ngày** |

### Bước 2: Cross-Region Replication (CRR)

> ⚠️ Cần bật **Versioning trên cả 2 Bucket**.

**Bucket đích (Destination):**

| Mục | Giá trị |
|-----|---------|
| Tên | `lab3-[クラス名+番号]-bucket2` |
| Region | **us-east-2 (Ohio)** |
| Versioning | Enable |

**Tạo Replication Rule:**

1. **Source bucket → Management → Replication rules → Create rule**

| Mục | Giá trị |
|-----|---------|
| Rule name | `[クラス名+番号]_rep01` |
| Status | Enabled |
| Scope | **All objects in bucket** |
| Destination | `lab3-[クラス名+番号]-bucket2` |
| IAM role | **CafeRole** (có sẵn) |

2. Nhấn **Save**
3. Upload file mới vào source bucket → sau vài phút thấy file xuất hiện ở bucket đích ✅

## Các Storage Class S3

| Class | Mô tả | Use case |
|-------|-------|----------|
| S3 Standard | Truy cập thường xuyên | Web, apps |
| S3 Standard-IA | Ít truy cập, cần nhanh | Backup |
| S3 One Zone-IA | 1 AZ, rẻ hơn | Dữ liệu tái tạo được |
| S3 Glacier Instant | Archive, lấy trong ms | Dữ liệu lâu dài |
| S3 Glacier Flexible | Archive, lấy trong giờ | Backup lâu dài |
| S3 Deep Archive | Rẻ nhất, lấy trong 12h+ | Lưu trữ hàng thập kỷ |
